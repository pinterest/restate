## RT0025

The maximum nesting depth for the futures (promises) an invocation awaits on has been reached.

Restate futures can be composed with combinators (e.g. `all`, `race`), and each level
of composition adds one level of nesting. This error is raised when an invocation awaits
on (or suspends on) a future whose nesting depth exceeds the limit configured in
`worker.invoker.max-awaited-future-depth` (default: `1000`).

This is most likely caused by unbounded or runaway recursion in the service code building
ever-deeper combinator futures, and it can also indicate a bug in the SDK.

When this limit is reached, the invocation is **paused** rather than retried or failed,
since retrying would deterministically hit the same limit again. You can resume the
invocation manually after addressing the root cause.

Suggestions:

* Check your service code for recursive or loop-based composition of futures (e.g. repeatedly
  wrapping futures in `all`/`race` combinators) that grows without a bound.
* If your code legitimately needs deeper nesting, increase `worker.invoker.max-awaited-future-depth`
  accordingly.

> **Warning:** Raise `worker.invoker.max-awaited-future-depth` with great care. Deeply nested
> futures are processed recursively (e.g. when encoding/decoding the awaited future set), so a
> large limit can exhaust the thread's stack and cause a stack overflow, which crashes the server
> and can destabilize the whole node. Prefer fixing the runaway recursion in your service code over
> raising this limit, and only increase it by a modest amount when strictly necessary.
